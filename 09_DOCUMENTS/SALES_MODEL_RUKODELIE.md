## КЛЮЧЕВОЕ ПРАВИЛО

> **Товар никогда не снимается с продажи только потому, что остаток меньше полной упаковки.**  
> Если в наличии 43 страза из пачки 144 — товар в наличии. AI продаёт то, что есть.

---

## ЗАЧЕМ НУЖНА МОДЕЛЬ ПРОДАЖ

Rukodelie.kz продаёт один и тот же товар двумя способами одновременно:

| Способ | Пример |
|--------|--------|
| **Целыми упаковками/рулонами** | Пачка страз 1440 шт., рулон бархотки 91 м |
| **В розницу (отрезом/поштучно)** | 10 страз, 0.5 м бархотки |

Без явного описания этой логики AI Seller либо продаёт неправильно ("у нас 65349 штук страз"), либо отказывает в продаже когда не должен.

## ЧЕТЫРЕ НОВЫХ ПОЛЯ

### `base_unit` — базовая единица хранения

Описывает, в чём физически хранится товар на складе.

| Значение | Смысл | Примеры товаров |
|----------|-------|----------------|
| `piece` | Штучный товар | Броши, пуговицы, пяльцы, молнии |
| `skein` | Моток | Пряжа всех видов |
| `meter` | Метр (отрезается от рулона) | Бархотка, бахрома, кружево, фатин, ткань |
| `roll` | Целый рулон | Рулоны ленты, упакованные тюки ткани |
| `pack` | Упаковка с несколькими единицами | Стразы, бисер, иглы, люверсы |
| `set` | Набор (неделимый) | Наборы баночек, наборы крючков |
| `kg` | Вес | Пух, наполнитель |

---

### `sale_mode` — режим продажи

Описывает, **как** можно купить товар.

| Значение | Смысл | Примеры товаров |
|----------|-------|----------------|
| `piece` | Только поштучно | Броши, пуговицы, пяльцы |
| `meter` | Только метрами | (нет в текущем каталоге) |
| `skein` | Только мотками | (нет в текущем каталоге) |
| `roll` | Только целым рулоном | (нет в текущем каталоге) |
| `pack` | Только целой упаковкой | Наборы, неделимые комплекты |
| `both` | И поштучно/отрезом И целиком | Стразы, бисер, пряжа, ленты, бахрома |

---

### `min_sale_qty` — минимальное количество для розничной продажи

Наименьшее количество, которое можно купить в розницу.

| Категория | min_sale_qty | Единица |
|-----------|-------------|---------|
| Бархотка | 0.2 | м |
| Бахрома | 0.5 | м |
| Стразы Swarovski | 1 | упаковка |
| Пряжа | 1 | моток |
| Бисер | 1 | упаковка |
| Броши, пуговицы | 1 | шт |

---

### `pack_qty` — количество базовых единиц в упаковке/рулоне

| Товар | pack_qty | Смысл |
|-------|---------|-------|
| Стразы ss6–ss20 | 1440 | 1440 страз в одном пакете |
| Стразы ss30 | 360 | 360 страз в пакете |
| Стразы ss34, ss40 | 144 | 144 страза в пакете |
| Стразы ss48 | 72 | 72 страза в пакете |
| Стразы 2855 8mm | 216 | 216 страз в пакете |
| Иглы | 10 | 10 игл в упаковке |
| Блочки/Люверсы | 100 | 100 штук в упаковке |
| Бархотка | 91 | 91 метр на рулоне |

## КАК ПРАВИЛЬНО ХРАНИТЬ ОСТАТОК

### Правило хранения `stock`

**Stock всегда хранится в базовых единицах (`base_unit`).**

| base_unit | Что хранится в `stock` |
|-----------|----------------------|
| `pack` | Количество **отдельных единиц** внутри всех упаковок |
| `roll` | Количество **метров** на всех рулонах |
| `skein` | Количество **мотков** |
| `piece` | Количество **штук** |
| `set` | Количество **наборов** |

**Пример:** стразы ss10, `stock = 65 349` означает 65 349 отдельных страз на складе.

### Как AI Seller рассчитывает наличие

```
full_packs  = FLOOR(stock / pack_qty)     # полных упаковок
loose_units = stock MOD pack_qty          # остаток не в упаковках
can_sell    = stock >= min_sale_qty       # можно ли продать вообще
```

**Пример для стразов ss34 (stock=43, pack_qty=144, min_sale_qty=1 упак):**
```
full_packs  = FLOOR(43 / 144) = 0
loose_units = 43 MOD 144 = 43
can_sell    = True (sale_mode=both → продаём россыпью)
```
AI Seller: "Полных упаковок нет, но в наличии 43 штуки — могу продать поштучно."

**Пример для бархотки (stock=273, unit=м, min_sale_qty=0.2 м):**
```
full_rolls  = FLOOR(273 / 91) = 3 рулона
loose_m     = 273 MOD 91 = 0 м
can_sell    = True (273 м ≥ 0.2 м)
```
AI Seller: "В наличии 273 метра. Минимальный отрез 20 см."

## ИСПРАВЛЕНИЕ ОШИБКИ В ПРАВИЛАХ НОРМАЛИЗАЦИИ v1.0

> Правило О3 из `DATA_NORMALIZATION_RULES.md` v1.0 было НЕВЕРНЫМ:  
> `ЕСЛИ stock_normalized = 0 → status = "discontinued"` — **УДАЛЕНО**

**Правильная логика:**

| Ситуация | Было (v1.0) | Стало (v1.1) |
|----------|-------------|-------------|
| stock < pack_qty | Снять с продажи | Продолжать продавать поштучно/отрезом |
| stock = 0 | Снять с продажи | Снять с продажи ✅ |
| sale_mode = pack И full_packs = 0 | Снять с продажи | Предупредить: "целой упаковки нет" |

## ВЛИЯНИЕ НА AI SELLER

### Сценарий 1: Клиент спрашивает о наличии страз ss34

**Без sale_mode (старая логика):**
> "Стразы ss34 в наличии нет." ❌ (stock=43, но 43 < 144 = ошибочный вывод)

**С sale_mode = both (новая логика):**
> "Стразы ss34 в наличии: 43 штуки. Полной упаковки (144 шт.) нет, но могу продать поштучно. Сколько вам нужно?" ✅

---

### Сценарий 2: Клиент хочет купить бархотку

**Без sale_mode:**
> "Бархотка в наличии 273 единицы." ❌ (непонятно — штуки или метры?)

**С sale_mode = both, base_unit = roll:**
> "Бархотка 12 мм в наличии 228 метров. Цена 160 ₸/метр. Минимальный отрез 20 см. Нужен целый рулон (91 м) или отрежем нужное количество?" ✅

---

### Сценарий 3: Клиент хочет пряжу

**С sale_mode = both, base_unit = skein:**
> "Пряжа YarnArt Wooden Club в наличии 29 мотков. Цена 3190 ₸/моток. Берёте один или несколько?" ✅

### Что изменилось в логике AI Seller

| Поле | Старый ответ | Новый ответ |
|------|-------------|-------------|
| `in_stock` | True/False | True если stock > 0 |
| Текст про наличие | "в наличии N единиц" | "N мотков / N метров / N штук / N упаковок" |
| Предложение купить | Одна опция | Два варианта если sale_mode = both |
| Отказ в продаже | При stock < pack_qty | Только при stock = 0 или stock < min_sale_qty |

## ОБНОВЛЁННАЯ СВОДНАЯ ТАБЛИЦА НОВЫХ ПОЛЕЙ

| Поле | Тип | Обяз. | Источник | Пример |
|------|-----|-------|----------|--------|
| `base_unit` | ENUM | Да | Вручную по категории | `pack`, `roll`, `skein` |
| `sale_mode` | ENUM | Да | Вручную по категории | `both`, `piece`, `pack` |
| `min_sale_qty` | DECIMAL | Нет | Сайт + вручную | `0.2`, `1`, `0.5` |
| `pack_qty` | INTEGER | Нет | Сайт (парсинг) | `1440`, `91`, `10` |
