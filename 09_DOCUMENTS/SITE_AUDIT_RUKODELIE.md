## КРАТКИЙ ВЫВОД

**Сайт Rukodelie.kz подходит для MVP. Рекомендован как основной источник каталога до подключения 1С.**

Карточки товаров содержат все ключевые данные: название, бренд, артикул, цену, точный остаток, характеристики, описание, фотографии и категорию. На страницах присутствует структурированный JSON-LD — это делает автоматический сбор данных надёжным и быстрым. Защита от парсинга минимальная.

## 3. СПОСОБЫ СБОРА ДАННЫХ

### Вариант 1: Через Sitemap ✅ РЕКОМЕНДУЕТСЯ

```
robots.txt → ссылка на sitemap.xml
sitemap.xml → 300+ URL товаров
каждый URL → парсинг карточки товара
```

**Плюсы:** самый полный охват, даёт все товары сразу, не нужно обходить категории.  
**Минусы:** sitemap может быть не полным (обрезан на ~350 URL при просмотре).  

### Вариант 2: Через страницы категорий ✅ ДОПОЛНИТЕЛЬНЫЙ

```
12 категорий (известны ID) → страницы листинга
пагинация по страницам → ссылки на карточки
карточки товаров → данные
```

**Плюсы:** гарантированно покрывает все товары по категориям.  
**Минусы:** нужно обходить пагинацию, дольше.

### Вариант 3: Через JSON-LD на карточках товаров ✅ ЛУЧШИЙ ПАРСЕР

Каждая страница товара содержит `<script type="application/ld+json">` с структурированными данными: название, бренд, цена, наличие, производитель. Это официальный машиночитаемый формат — парсить надёжнее, чем HTML.

### Вариант 4: Через API ❌ НЕДОСТУПЕН

OpenCart имеет REST API, но на Rukodelie.kz он не подключён или закрыт. Ответ 404 на `/index.php?route=api/product/info`.

### Вариант 5: Парсинг HTML ✅ РЕЗЕРВНЫЙ

Стандартная HTML-структура OpenCart позволяет парсить классы CSS. Работает, но хрупко — ломается при обновлении шаблона.

## 4. РИСКИ

| Риск | Уровень | Описание |
|------|---------|----------|
| Защита от парсинга | 🟢 Низкий | Нет CAPTCHA, нет Cloudflare, нет rate-limit по факту |

## 6. РЕКОМЕНДУЕМЫЙ СЛЕДУЮЩИЙ ШАГ

**Запустить пилотный сбор данных по sitemap для 50–100 товаров вручную или с помощью простого скрипта.**

### Конкретный план:

1. Скачать `sitemap.xml` полностью
2. Извлечь все URL товаров (паттерн: `rukodelie.kz/[slug]_[ID]`)
3. Для каждого URL — открыть страницу и скопировать данные из JSON-LD + остаток
4. Сохранить в `02_PRODUCT_DATABASE/catalog_raw.csv`
5. Владелец проверяет 10 позиций вручную на актуальность цен и остатков

**Файл:** `02_PRODUCT_DATABASE/catalog_raw.csv`

### Структура файла

```csv

```

### Этапы заполнения

| Этап | Действие | Кто | Срок |
|------|----------|-----|------|
| 1 | Скачать sitemap.xml и извлечь URL | AI PM | 1 день |
| 2 | Парсинг 50 приоритетных товаров (пряжа, фурнитура) | AI PM | 2–3 дня |
| 3 | Проверка 10 позиций вручную на актуальность | Владелец | 1 день |

**Итого до готового каталога: ~7–10 дней при наличии подтверждения от владельца.**

### Приоритетные категории для MVP

2. Нитки
3. Молнии
