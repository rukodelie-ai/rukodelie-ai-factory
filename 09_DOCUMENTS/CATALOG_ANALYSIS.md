## 3. ПОЛЯ, ТРЕБУЮЩИЕ НОРМАЛИЗАЦИИ

### 3.1 Поле `brand` — 3 проблемы

**Проблема 1: дублирование написания**
```
"Гамма" (id: 24, 46) и "Gamma" (id: 35) → нормализовать в "Gamma"
```

**Проблема 2: страна вместо бренда**
```
id 29: brand = "Россия" → brand = "-", country = "Россия"
```

**Проблема 3: пустые значения**
```
16 товаров: brand = "-"
Причины: товары без бренда на сайте / нужна ручная доработка
```

---

### 3.2 Поле `article` — 3 несовместимых формата

| Формат | Примеры | Кол-во |
|--------|---------|--------|
| Числовой с ведущими нулями | `0000034556`, `00000000251` | ~20 |
| Дефисный внешний | `00-00003382`, `00-00002958` | ~15 |
| Внутренний RD-код | `8/3/5/2-№80`, `54/3/2-`, `20/44/1-` | ~15 |

**Проблема:** один и тот же товар может иметь оба формата. В CSV сохранён только один.  
**Решение:** добавить поле `article_rd` для внутреннего кода RD.

---

### 3.3 Поле `stock` — смешаны единицы измерения

| Тип товара | Единица | Кол-во товаров |
|------------|---------|---------------|
| Штучные (стразы, иглы, броши...) | шт | 42 |
| Метражные (бархотка, бахрома) | метр | 8 |

**Проблема:** поле `stock` содержит числа в разных единицах без маркировки.  
Пример: stock=273 у Бархотки 10мм — это 273 **метра**, а не 273 штуки.  
**Решение:** добавить поле `unit` (шт / м / г).

---

### 3.4 Поле `category` — нестабильная глубина

| Глубина | Пример | Кол-во |
|---------|--------|--------|
| 2 уровня | `Украшения для волос > Бижутерия` | 3 |
| 3 уровня | `SWAROVSKI > Стразы > Горячей фиксации 2038` | 47 |

**Проблема:** одно текстовое поле хранит всю иерархию — сложно фильтровать.  
**Решение:** разбить на `category_l1`, `category_l2`, `category_l3`.

---

### 3.5 Поле `description` — непоследовательный формат

Одни описания включают состав и страну производства прямо в текст, другие — нет.  
Нет стандарта длины: от 40 до 120 символов.  
**Решение:** выделить отдельные поля: `composition`, `country`, `size`.

## 5. ОКОНЧАТЕЛЬНАЯ СТРУКТУРА PRODUCT DATABASE v1

### Принципы

1. **Единица хранения — SKU** (одна запись = один товар, вне зависимости от цвета)
2. **Категория — трёхуровневая** в отдельных полях
3. **Артикул — первичный ключ**, уникален в рамках базы
4. **Единица измерения** обязательна для корректной работы AI Seller
5. **Статус** контролирует видимость товара для агентов

### Полная схема CSV (Product Database v1)

```
# БЛОК 1: ИДЕНТИФИКАЦИЯ
id              — порядковый номер в базе
article         — основной артикул (уникальный ключ)
article_rd      — внутренний RD-код сайта (вторичный)
status          — draft | ready | published | discontinued

# БЛОК 2: КЛАССИФИКАЦИЯ
name            — полное коммерческое название
brand           — бренд (нормализованный)
category_l1     — главная категория
category_l2     — подкатегория
category_l3     — подподкатегория (может быть пустым)

# БЛОК 3: ХАРАКТЕРИСТИКИ ТОВАРА
size            — размер (ss10, 10мм, 3*3см и т.д.)
composition     — состав (55% хлопок, 45% акрил)
country         — страна производства
weight_g        — вес единицы, грамм
colors_available — список цветов через ";"
qty_per_pack    — кол-во в упаковке

# БЛОК 4: КОММЕРЦИЯ
price_kzt       — розничная цена в тенге
stock           — остаток
unit            — шт | м | г | рулон
in_stock        — true | false (вычисляемый: stock > 0)
min_order_retail   — минимальный заказ розница
min_order_wholesale — минимальный заказ опт

# БЛОК 5: КОНТЕНТ
description_short — краткое описание (1-2 предложения)
description_full  — полное описание с сайта
seo_description   — SEO-текст (генерируется AI)
keywords          — ключевые слова (генерируются AI)
photo_url         — URL главного фото

# БЛОК 6: СЛУЖЕБНЫЕ
url             — ссылка на сайте
date_parsed     — дата сбора данных с сайта
date_added      — дата добавления в базу
date_updated    — дата последнего обновления
```
