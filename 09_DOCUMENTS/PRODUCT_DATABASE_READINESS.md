## 2. ПОЛЯ, КОТОРЫЕ ПРИДЁТСЯ БРАТЬ ИЗ 1С

### Для MVP — из 1С ничего не требуется

Все 33 поля схемы v1 покрываются сайтом, правилами или AI-генерацией. 1С понадобится только для следующих расширений (за пределами v1):

| Поле (будущее) | Почему из 1С | Этап |
|----------------|-------------|------|
| `purchase_price` | Закупочная цена — никогда не на сайте | v2 |
| `cost_price` | Себестоимость — бухгалтерские данные | v2 |
| `supplier_id` | Поставщик — не на сайте | v2 |
| `sales_history` | История продаж — только транзакционная БД | v2 |
| `wholesale_price_tiers` | Оптовые ценовые градации | v2 |

### Важная оговорка по точности

| Поле | Есть на сайте | Риск расхождения с 1С |
|------|--------------|----------------------|
| `stock` | ✅ Да | Средний — сайт обновляется при продаже, но возможна задержка |
| `price_kzt` | ✅ Да | Низкий — цены обычно синхронны |
| `min_order_retail` | ✅ Да | Низкий |

**Рекомендация:** для MVP использовать данные с сайта. При интеграции с 1С в v2 — синхронизировать `stock` и `price_kzt` в приоритете.

### Два уровня готовности

**Уровень 1 — "Может отвечать" (сейчас, без доработок): 61%**
AI Seller может:
- отвечать на вопросы о наличии и цене ✅
- описывать товары текстом ✅
- находить товары по категории (после split) ✅
- называть бренд для 64% товаров ✅

AI Seller не может:
- показывать фото ❌
- отвечать на вопросы о цветах ❌
- правильно называть минимальный заказ ❌
- корректно работать с моделью pack/both ❌

**Уровень 2 — "Полноценный MVP" (после второго парсинга, ~7 дней): 94%**
Добавится:
- photo_url → бот показывает фото
- colors_available → ответы на вопросы о цвете
- min_order_retail / min_sale_qty → правильные условия
- description_full → SEO-тексты
- pack_qty → точная модель продаж страз и лент

Остаток 6% — `status` и ручное заполнение brand — 60–70 минут работы владельца.
