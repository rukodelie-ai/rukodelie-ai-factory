## БЛОК 1: ИДЕНТИФИКАЦИЯ

### 1. `id`

| Параметр | Значение |
|----------|---------|
| Тип данных | INTEGER (автоинкремент) |
| Обязательное | Да |
| Источник | Генерируется автоматически |
| Пример | `1`, `42`, `500` |

**Зачем AI-продавцу:** не используется в диалоге напрямую. Нужен как внутренний ключ для ссылок между записями и логирования запросов.

---

### 2. `article`

| Параметр | Значение |
|----------|---------|
| Тип данных | STRING (max 30 символов) |
| Обязательное | Да — уникальный первичный ключ |
| Источник | Сайт (парсинг карточки товара) |
| Пример | `00-00003382`, `0000034556` |

**Зачем AI-продавцу:** главный идентификатор при поиске товара в базе. Связывает запись с сайтом и будущей интеграцией с 1С. Если клиент называет артикул — AI сразу находит нужную запись.

---

### 3. `article_rd`

| Параметр | Значение |
|----------|---------|
| Тип данных | STRING (max 20 символов) |
| Обязательное | Нет |
| Источник | Сайт (внутренний RD-код, парсинг) |
| Пример | `8/3/5/2-№80`, `20/44/1-`, `54/3/2-` |

**Зачем AI-продавцу:** резервный поиск, если основной артикул не найден. При синхронизации с сайтом этот код позволяет обновлять запись без ошибок дублирования.

---

### 4. `status`

| Параметр | Значение |
|----------|---------|
| Тип данных | ENUM: `draft` \| `ready` \| `published` \| `discontinued` |
| Обязательное | Да |
| Источник | Вручную (владелец или AI PM) |
| Пример | `published` |

**Зачем AI-продавцу:** главный фильтр видимости. AI Seller показывает клиентам только товары со статусом `published`. Товары `discontinued` не предлагаются, даже если запрос совпадает. Товары `draft` — невидимы для клиента.

### 5. `name`

|----------|---------|

**Зачем AI-продавцу:** основной текст, который произносится при ответе клиенту. "У нас есть **Пряжа Alpaca Gold Paillettes YarnArt** за 1450 ₸."

### 6. `brand`

**Зачем AI-продавцу:** фильтрация по бренду ("покажи всё от YarnArt"), доверие клиента ("это оригинальный Swarovski из Австрии"), up-sell ("есть похожее от того же бренда"). Требует нормализации: `Гамма` и `Gamma` → один бренд.

---

### 8. `category_l2`

**Зачем AI-продавцу:** максимально точная классификация для профессиональных запросов ("стразы 2038, а не 2058"). Без этого поля невозможно различить схожие подгруппы в категории SWAROVSKI.

## БЛОК 4: КОММЕРЦИЯ

### 15. `price_kzt`

| Параметр | Значение |
|----------|---------|
| Тип данных | INTEGER (тенге, без копеек) |
| Обязательное | Да |
| Источник | Сайт (парсинг, реальное время) |
| Пример | `1450`, `60`, `4200` |

**Зачем AI-продавцу:** главный ответ на вопрос "Сколько стоит?". Требует регулярного обновления — цены на сайте могут меняться. Хранить как целое число (тенге без дробей).

---

### 16. `stock`

| Параметр | Значение |
|----------|---------|
| Тип данных | DECIMAL (10, 2) |
| Обязательное | Да |
| Источник | Сайт (парсинг остатка в реальном времени) |
| Пример | `7`, `398`, `273.5` (для метражных товаров) |

**Зачем AI-продавцу:** точное количество для ответа "Есть ли 50 штук?" или "Сколько метров осталось?". DECIMAL нужен для метражных товаров (273.5 м). Связан с полем `unit`.

---

### 17. `unit`

| Параметр | Значение |
|----------|---------|
| Тип данных | ENUM: `шт` \| `м` \| `г` \| `рулон` \| `упаковка` |
| Обязательное | Да |
| Источник | Вручную (при создании карточки) + парсинг |
| Пример | `шт` (иглы), `м` (бархотка), `упаковка` (стразы) |

**Зачем AI-продавцу:** критически важное поле. Без него невозможно корректно ответить на вопрос о цене и наличии. Пример ошибки без `unit`: "бархотка стоит 130 ₸, в наличии 273" — неясно, 273 штуки или 273 метра. С полем: "130 ₸ за метр, в наличии 273 метра."

---

### 18. `in_stock`

| Параметр | Значение |
|----------|---------|
| Тип данных | BOOLEAN (`true` \| `false`) |
| Обязательное | Да (вычисляемое) |
| Источник | Вычисляется: `stock > 0` → `true` |
| Пример | `true`, `false` |

**Зачем AI-продавцу:** быстрый фильтр "есть/нет" без математики. Позволяет мгновенно исключить товары без остатка из выдачи. Первый шаг любого поиска — отфильтровать по `in_stock = true`.

---

### 19. `min_order_retail`

| Параметр | Значение |
|----------|---------|
| Тип данных | DECIMAL (10, 2) |
| Обязательное | Нет (но важное) |
| Источник | Сайт (парсинг условий продажи) |
| Пример | `1` (шт), `0.5` (м), `5` (упаковок) |

**Зачем AI-продавцу:** "Можно купить 1 штуку?" — без этого поля AI не может ответить. Особенно важно для метражных товаров (минимум 0.2 м или 0.5 м) и страз (минимум 5 упаковок).

---

### 20. `min_order_wholesale`

| Параметр | Значение |
|----------|---------|
| Тип данных | DECIMAL (10, 2) |
| Обязательное | Нет |
| Источник | Сайт (парсинг условий продажи) |
| Пример | `10` (шт), `50` (упаковок), `45.5` (м) |

**Зачем AI-продавцу:** при определении оптового клиента — сразу называть условия оптовой закупки и направлять к соответствующему менеджеру или давать скидку. Позволяет AI квалифицировать лид как розничный или оптовый.

---

### 21. `base_unit`

| Параметр | Значение |
|----------|---------|
| Тип данных | ENUM: `piece` \| `skein` \| `meter` \| `roll` \| `pack` \| `set` \| `kg` |
| Обязательное | Да |
| Источник | Вручную по категории товара |
| Пример | `pack` (стразы), `skein` (пряжа), `roll` (бархотка, бахрома) |

**Зачем AI-продавцу:** базовая единица, в которой физически хранится товар на складе. Определяет смысл числа в поле `stock`. Без этого поля AI не знает, что `stock=65349` у страз — это отдельные кристаллы, а не пачки.

---

### 22. `sale_mode`

| Параметр | Значение |
|----------|---------|
| Тип данных | ENUM: `piece` \| `meter` \| `skein` \| `roll` \| `pack` \| `both` |
| Обязательное | Да |
| Источник | Вручную по категории товара |
| Пример | `both` (стразы, ленты, пряжа), `piece` (броши, пуговицы) |

**Зачем AI-продавцу:** режим `both` означает — товар продаётся и поштучно/отрезом, и целыми упаковками/рулонами. AI сам предлагает оба варианта и никогда не отказывает в продаже только потому, что нет полной упаковки. Это правило распространяется на стразы, бисер, пряжу, ленты.

---

### 23. `min_sale_qty`

| Параметр | Значение |
|----------|---------|
| Тип данных | DECIMAL (10, 2) |
| Обязательное | Нет |
| Источник | Сайт (парсинг) + вручную |
| Пример | `0.2` (м бархотки), `0.5` (м бахромы), `1` (упаковка страз) |

**Зачем AI-продавцу:** минимальный объём розничной продажи. Если клиент запрашивает меньше — AI корректно отвечает: "Минимальный отрез 20 см." Важнее поля `min_order_retail` — учитывает модель продажи.

---

### 24. `pack_qty`

| Параметр | Значение |
|----------|---------|
| Тип данных | INTEGER |
| Обязательное | Нет |
| Источник | Сайт (парсинг) |
| Пример | `1440` (стразы ss10), `91` (метров на рулоне бархотки), `10` (иглы) |

**Зачем AI-продавцу:** количество единиц в одной упаковке или рулоне. AI вычисляет на лету: `full_packs = FLOOR(stock / pack_qty)` и сообщает клиенту "в наличии 45 упаковок и ещё 9 штук россыпью".

## БЛОК 5: КОНТЕНТ

### 25. `description_short`

| Параметр | Значение |
|----------|---------|
| Тип данных | TEXT (max 300 символов) |
| Обязательное | Да |
| Источник | Сайт (парсинг) + ручная правка |
| Пример | `Клеевые стразы горячей фиксации ss10 (2.7-2.8 мм). Австрия. 1440 шт. в упаковке.` |

**Зачем AI-продавцу:** компактный ответ в чате — когда клиент спрашивает "что это за товар?". Умещается в одно сообщение Telegram. Не должен превышать 300 символов для корректного отображения в превью.

---

### 22. `description_full`

| Параметр | Значение |
|----------|---------|
| Тип данных | TEXT (без ограничения) |
| Обязательное | Нет |
| Источник | Сайт (парсинг полного текста описания) |
| Пример | Полный текст с сайта, 200-500 слов |

**Зачем AI-продавцу:** входной контекст для генерации ответов на сложные вопросы. AI читает полное описание и формулирует ответ своими словами. Также используется SEO Agent как основа для генерации SEO-текстов.

---

### 23. `seo_description`

| Параметр | Значение |
|----------|---------|
| Тип данных | TEXT (150-300 слов) |
| Обязательное | Нет |
| Источник | Генерируется SEO Agent (AI) |
| Пример | "Стразы Swarovski ss10 купить в Алматы..." |

**Зачем AI-продавцу:** публикация на маркетплейсах (Kaspi, OLX) и на сайте. SEO Agent генерирует по шаблону из `10_PROMPTS`. Не используется в живом диалоге — только для публикации.

---

### 24. `keywords`

| Параметр | Значение |
|----------|---------|
| Тип данных | STRING (список через `;`) |
| Обязательное | Нет |
| Источник | Генерируется SEO Agent (AI) |

**Зачем AI-продавцу:** поиск товаров по синонимам и альтернативным запросам клиента. "Стразики" → находит "стразы". "Блёстки на ткань" → находит "стразы клеевые". Расширяет покрытие поиска.

---

### 25. `photo_url`

**Зачем AI-продавцу:** отправка фото в Telegram по запросу клиента "покажи фото". Без этого поля бот может отвечать только текстом. Фото критически повышает конверсию в покупку.

## БЛОК 6: СЛУЖЕБНЫЕ

### 27. `date_parsed`

| Параметр | Значение |
|----------|---------|
| Тип данных | DATE (`YYYY-MM-DD`) |
| Обязательное | Да |
| Источник | Автоматически при парсинге |
| Пример | `2026-06-07` |

**Зачем AI-продавцу:** контроль актуальности данных. Если `date_parsed` старше 7 дней — цена и остаток могут быть устаревшими. Analytics Agent может алертить владельца: "14 товаров не обновлялись более недели."

---

### 28. `date_added`

| Параметр | Значение |
|----------|---------|
| Тип данных | DATE (`YYYY-MM-DD`) |
| Обязательное | Да |
| Источник | Автоматически при создании записи |
| Пример | `2026-06-07` |

**Зачем AI-продавцу:** не используется в диалоге. Нужен Analytics Agent для анализа роста каталога ("за июнь добавлено 120 товаров") и Marketing Agent для кампании "новинки недели".

---

### 29. `date_updated`

| Параметр | Значение |
|----------|---------|
| Тип данных | DATE (`YYYY-MM-DD`) |

**Зачем AI-продавцу:** не используется в диалоге. Нужен для контроля версий данных и аудита: кто и когда менял цену, остаток или описание. Защита от случайных перезаписей при повторном парсинге.
