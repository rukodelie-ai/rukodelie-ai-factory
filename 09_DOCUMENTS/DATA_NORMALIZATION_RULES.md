## ПРАВИЛО 1: НОРМАЛИЗАЦИЯ БРЕНДОВ

### 1.1 Привести написание к единому стандарту

**Логика:** один бренд — одно написание. Приоритет — официальное латинское или русское название производителя.

| Исходное значение | Исправленное значение | Логика |
|-------------------|----------------------|--------|
| `Гамма` | `Gamma` | Официальное название бренда — латиница. Встречается у id 24, 46 |
| `Gamma` | `Gamma` | Уже корректно. id 35 |
| `Арти` | `Арти` | Оставить как есть — русский бренд. Унифицировать регистр: первая заглавная |
| `АРТИ` | `Арти` | Привести к единому регистру |
| `Swarovski` | `Swarovski` | Корректно. Не трогать |
| `YarnArt` | `YarnArt` | Корректно. Не трогать |
| `Preciosa` | `Preciosa` | Корректно. Не трогать |
| `Angelica Fashion` | `Angelica Fashion` | Корректно. Не трогать |
| `BLITZ` | `Gamma` | BLITZ — серия/модель изделия, не бренд. Производитель пяльцев — Gamma. Переименовать, модель перенести в поле `name` или `size` |

**Итоговый словарь брендов:**
```
Gamma, Арти, Swarovski, YarnArt, Preciosa, Angelica Fashion, BLITZ (серия Gamma)
```

### 1.2 Страна производства ошибочно попала в поле brand

**Логика:** поле `brand` должно содержать только название производителя. Страна — в поле `country`.

| id | Исходное brand | Исправленное brand | Значение country |
|----|---------------|-------------------|-----------------|
| 29 | `Россия` | `-` (пусто) | `Россия` |

**Правило автоматического определения:**
```
ЕСЛИ brand IN ("Россия", "Турция", "Китай", "ОАЭ", "Австрия", "Чехия", "Корея")
ТО: country = brand, brand = ""
```

### 1.3 Бренд не указан

**Логика:** значение `"-"` означает отсутствие данных. Заменить на пустую строку для корректной фильтрации.

| Исходное значение | Исправленное значение | Логика |
|-------------------|----------------------|--------|
| `-` | `` (пусто) | Пустая строка — стандартное значение "нет данных" |

**Затронуто:** ids 21, 23, 26, 27, 28, 30–34, 36–37, 41–44, 47–50 = **18 записей**

**Правило:**
```
ЕСЛИ brand = "-" ТО brand = ""
```

## ПРАВИЛО 2: НОРМАЛИЗАЦИЯ КАТЕГОРИЙ

### 2.1 Разбить поле category на три уровня

**Логика:** поле `category` хранит путь через ` > `. Необходимо разбить на отдельные поля `category_l1`, `category_l2`, `category_l3`.

| Исходное category | category_l1 | category_l2 | category_l3 |
|-------------------|-------------|-------------|-------------|
| `Вязание > Пряжа > YarnArt` | `Вязание` | `Пряжа` | `YarnArt` |
| `Молнии > Бегунки для молний` | `Молнии` | `Бегунки для молний` | *(пусто)* |
| `Подарочные и упаковочные товары` | `Подарочные и упаковочные товары` | *(пусто)* | *(пусто)* |

**Правило:**
```
parts = category.split(" > ")
category_l1 = parts[0] если есть, иначе ""
category_l2 = parts[1] если есть, иначе ""
category_l3 = parts[2] если есть, иначе ""
```

## ПРАВИЛО 4: НОРМАЛИЗАЦИЯ ОСТАТКОВ

### 4.1 Хранение остатков — без пересчёта

**Правило (обновлено в v1.1):** `stock` хранится как количество **базовых единиц** (`base_unit`) на складе. Пересчёт в упаковки НЕ производится.

> **Причина:** товар никогда не снимается с продажи только потому, что остаток меньше полной упаковки. Если `sale_mode = both` — товар продаётся и поштучно, и упаковками. См. `SALES_MODEL_RUKODELIE.md`.

**Как AI Seller интерпретирует stock:**

```
full_packs  = FLOOR(stock / pack_qty)   # для информации клиенту
loose_units = stock MOD pack_qty        # остаток вне упаковок
can_sell    = stock > 0                 # товар доступен
```

### 4.3 Флаги низкого остатка (обновлено v1.1)

**Логика:** предупреждать AI Seller о нехватке товара, но не снимать с продажи автоматически.

| Условие | Флаг | Действие AI Seller |
|---------|------|-------------------|
| stock = 0 | `out_of_stock` | Не предлагать. Сообщить "нет в наличии" |
| stock > 0 И full_packs = 0 И sale_mode = `both` | `loose_only` | "Полной упаковки нет, есть N штук россыпью" |
| stock > 0 И full_packs = 0 И sale_mode = `pack` | `low_stock` | Предупредить владельца — заказать товар |
| 0 < full_packs ≤ 2 | `low_stock` | "Осталось мало — 1–2 упаковки" |
| 0 < stock ≤ 3 (штучный товар) | `low_stock` | "Осталось мало — X штук" |

**Правило:**
```
ЕСЛИ stock = 0 → in_stock = false (единственный случай снятия с продажи)
ЕСЛИ stock > 0 → in_stock = true (всегда, независимо от pack_qty)
full_packs = FLOOR(stock / pack_qty) — вычислять при каждом ответе AI
```

> ⚠️ **Отменённое правило v1.0:** `ЕСЛИ stock < pack_qty → status = discontinued` — **НЕВЕРНО. УДАЛЕНО.**

## ПРАВИЛО 5: НОРМАЛИЗАЦИЯ АРТИКУЛОВ

### 5.3 Стандартизация длины цифровых артикулов

**Проблема:** числовые артикулы имеют разную длину: 8, 10, 11 символов.

| Исходный | Длина | Нормализованный (10 цифр) |
|----------|-------|--------------------------|
| `0000034556` | 10 | `0000034556` |
| `00000000251` | 11 | `00000000251` — оставить как есть |
| `0000002990` | 10 | `0000002990` |
| `00000000008` | 11 | `00000000008` — оставить как есть |

**Вывод:** длина артикула не унифицирована на сайте. Не нормализовывать принудительно — риск коллизий. Сохранить "как есть" и использовать как строку, не как число.

| А3 | article | Хранить как STRING, не INTEGER | Тип данных | 50 |

## ПРАВИЛО 6: НОРМАЛИЗАЦИЯ МОДЕЛИ ПРОДАЖ

### 6.1 Заполнение base_unit по категории

```
base_unit = "pack"   ЕСЛИ category СОДЕРЖИТ "Стразы" ИЛИ "Бисер" ИЛИ "Иглы" ИЛИ "Люверсы"
base_unit = "skein"  ЕСЛИ category СОДЕРЖИТ "Пряжа"
base_unit = "roll"   ЕСЛИ category СОДЕРЖИТ "Бархотка" ИЛИ "Бахрома" ИЛИ "Лента" ИЛИ "Ткань"
base_unit = "set"    ЕСЛИ name СОДЕРЖИТ "набор" ИЛИ "Набор"
base_unit = "piece"  В ОСТАЛЬНЫХ СЛУЧАЯХ
```

### 6.2 Заполнение sale_mode по категории

```
sale_mode = "both"   ЕСЛИ base_unit IN ("pack", "roll", "skein")
sale_mode = "pack"   ЕСЛИ base_unit = "set"
sale_mode = "piece"  ЕСЛИ base_unit = "piece"
```

### 6.3 Заполнение min_sale_qty

```
min_sale_qty = 0.2   ЕСЛИ category СОДЕРЖИТ "Бархотка"
min_sale_qty = 0.5   ЕСЛИ category СОДЕРЖИТ "Бахрома" ИЛИ "Лента" ИЛИ "Кружево" ИЛИ "Фатин"
min_sale_qty = 0.2   ЕСЛИ category СОДЕРЖИТ "Ткань"
min_sale_qty = 1     В ОСТАЛЬНЫХ СЛУЧАЯХ
```

### 6.4 Заполнение pack_qty из описания

```
ЕСЛИ description СОДЕРЖИТ "1440 шт" → pack_qty = 1440
ЕСЛИ description СОДЕРЖИТ "360 шт"  → pack_qty = 360
ЕСЛИ description СОДЕРЖИТ "216 шт"  → pack_qty = 216
ЕСЛИ description СОДЕРЖИТ "144 шт"  → pack_qty = 144
ЕСЛИ description СОДЕРЖИТ "72 шт"   → pack_qty = 72
ЕСЛИ description СОДЕРЖИТ "100 шт"  → pack_qty = 100
ЕСЛИ description СОДЕРЖИТ "10 шт"   → pack_qty = 10
ЕСЛИ description СОДЕРЖИТ "91 м" ИЛИ "91 метр" → pack_qty = 91
ИНАЧЕ → pack_qty = NULL (заполнить вручную)
```

## ПОРЯДОК ПРИМЕНЕНИЯ ПРАВИЛ (обновлено v1.1)

Правила нужно применять строго в порядке зависимостей:

```
Шаг 1: Артикулы       (А1 → А2 → А3)
        — первым, чтобы не потерять данные

Шаг 2: Категории      (К1 → К2 → К3 → К4)
        — разбивка нужна до правил по unit и sale_mode

Шаг 3: Единицы        (Е1 → Е2 → Е3)
        — unit нужен до модели продаж

Шаг 4: Модель продаж  (М4 → М1 → М2 → М3)
        — pack_qty первым (нужен для min_sale_qty)

Шаг 5: Остатки        (О1 → О2 → О3)
        — только после установки base_unit и sale_mode

Шаг 5: Бренды     (Б5 → Б3 → Б4 → Б1 → Б2 → Б6)
        — сначала очистить мусор, потом заполнять из name
```
