## 1. ФУНКЦИИ MVP (ЧТО ВХОДИТ)

### Группа A — Обязательные (без них MVP не работает)

| # | Функция | Зачем |
|---|---------|-------|
| A1 | Приветствие + онбординг | Первый контакт, доверие, навигация |
| A2 | Поиск товара по запросу | Клиент пишет "пряжа" или "стразы" — AI находит |
| A3 | Показ карточки товара | Фото + цена + наличие + краткое описание |
| A4 | Ответ на "есть ли в наличии?" | Читает `stock`, `in_stock`, `sale_mode` |
| A5 | Ответ на "сколько стоит?" | Читает `price_kzt` + `unit` + `min_sale_qty` |
| A6 | Ответ на "какие цвета есть?" | Читает `colors_available` |
| A7 | Правильная модель продаж | sale_mode=both → предлагает оба варианта |
| A8 | Принятие заказа | Имя + телефон + товар + количество |
| A9 | Уведомление владельца | Алерт в admin-канал с данными заказа |

### Группа B — Важные (добавляют конверсию)

| # | Функция | Зачем |
|---|---------|-------|
| B1 | Показ фото товара | `photo_url` → конверсия x2 |
| B2 | Ответ на "из чего сделано?" | Читает `composition` (критично для пряжи) |
| B3 | Ответ на "какой минимальный заказ?" | Читает `min_order_retail` + `min_sale_qty` |
| B4 | Ссылка на товар на сайте | Читает `url` → клиент видит полную карточку |
| B5 | Кнопки быстрого выбора категории | UX — снижает трение при первом контакте |
| B6 | Повторный контакт (follow-up) | "Вы выбрали X — оформить заказ?" |

### Группа C — Желательные (если останется время)

| # | Функция | Зачем |
|---|---------|-------|
| C1 | Ответ на "что ещё подойдёт?" | Похожие товары из той же категории |
| C2 | Оптовый запрос | Квалификация: розница vs. опт |
| C3 | Команда /catalog | Показ списка категорий |
| C4 | Команда /help | Что умеет бот |

## 4. ДАННЫЕ ИЗ catalog_v1.csv

### Какие поля читает AI Seller для каждой функции

| Функция AI Seller | Поля из catalog_v1.csv |
|-------------------|------------------------|
| Поиск товара | `name`, `category_l1/l2/l3`, `brand`, `description_short`, `keywords` |
| Показ карточки | `name`, `price_kzt`, `unit`, `stock`, `in_stock`, `colors_available`, `size`, `composition` |
| Показ фото | `photo_url` |
| Ответ о наличии | `stock`, `in_stock`, `base_unit`, `sale_mode`, `pack_qty`, `min_sale_qty` |
| Цена и условия | `price_kzt`, `unit`, `min_order_retail`, `min_order_wholesale`, `sale_mode` |
| Ссылка на сайт | `url` |
| Состав материала | `composition`, `country` |
| Цвета | `colors_available` |
| Мин. заказ | `min_sale_qty`, `min_order_retail` |
| Фильтр: только опубликованные | `status = published` |

### Алгоритм ответа о наличии (на основе SALES_MODEL_RUKODELIE.md)

```
ЕСЛИ stock = 0:
    → "К сожалению, товара нет в наличии"

ЕСЛИ sale_mode = "piece":
    → "В наличии N штук. Цена X ₸/шт."

ЕСЛИ sale_mode = "both" И pack_qty > 0:
    full_packs = FLOOR(stock / pack_qty)
    loose = stock MOD pack_qty
    ЕСЛИ full_packs > 0:
        → "В наличии N упаковок по {pack_qty} шт. + ещё {loose} шт."
    ИНАЧЕ:
        → "Полной упаковки нет, но есть {loose} шт. — продаём поштучно"

ЕСЛИ base_unit = "roll":
    → "В наличии N метров. Цена X ₸/метр. Мин. отрез {min_sale_qty} м."
```

### Состояния диалога (FSM)

```
IDLE → GREETING → SEARCH → PRODUCT_VIEW → ORDER_START → ORDER_CONFIRM → DONE
         ↑                      ↓
         └──────── CLARIFY ←────┘
```

| Состояние | Действие AI | Данные |
|-----------|------------|--------|
| IDLE | Ждёт сообщения | — |
| GREETING | Приветствует, показывает категории | — |
| SEARCH | Ищет товары по запросу | catalog_v1.csv |
| PRODUCT_VIEW | Показывает карточку, отвечает на вопросы | catalog_v1.csv |
| ORDER_START | Запрашивает имя и телефон | — |
| ORDER_CONFIRM | Подтверждает заказ, отправляет алерт | Google Sheets |
| DONE | Предлагает продолжить покупки | — |

## 6. АРХИТЕКТУРА GOOGLE SHEETS

### Файл: RUKODELIE_AI_FACTORY (один Google Sheets документ)

---

### Лист 1: `catalog` (зеркало catalog_v1.csv)

**Назначение:** единственный источник данных для AI Seller. Владелец может вручную менять статус, цены, остатки прямо в таблице.

| Колонка | Источник | Кто редактирует |
|---------|---------|----------------|
| Все 30 полей из catalog_v1.csv | Импорт из CSV | AI PM (обновление) |
| `status` | Вручную | Владелец |
| `price_kzt` | Сайт / вручную | Владелец |
| `stock` | Сайт / вручную | Владелец |

**Фильтр по умолчанию:** `status = published` — AI Seller видит только эти строки.

**Обновление:** раз в 1-2 дня AI PM парсит сайт и обновляет `stock` и `price_kzt`.

---

### Лист 2: `orders` (журнал заказов)

| Колонка | Описание | Пример |
|---------|---------|--------|
| `order_id` | Авто-инкремент | #001 |
| `date` | Дата и время | 2026-06-07 15:30 |
| `client_name` | Имя из диалога | Айгуль |
| `client_phone` | Телефон | +7 777 123 45 67 |
| `product_id` | id товара | 5 |
| `product_name` | Название товара | Пряжа Wooden Club YarnArt |
| `quantity` | Количество + единица | 2 мотка |
| `color` | Выбранный цвет | Зелёный |
| `price_unit` | Цена на момент заказа | 3 190 ₸ |
| `total_price` | Итого | 6 380 ₸ |
| `order_status` | Статус | new / confirmed / delivered |
| `tg_chat_id` | ID чата клиента | 123456789 |
| `notes` | Комментарии | — |

**Автоматически:** при каждом новом заказе Google Sheets отправляет уведомление владельцу в Telegram.

---

### Лист 3: `settings` (настройки бота)

| Параметр | Значение | Описание |
|---------|---------|---------|
| `bot_active` | TRUE / FALSE | Включить/выключить бот |
| `admin_chat_id` | ID чата владельца | Для алертов |
| `greeting_text` | Текст приветствия | Редактируется вручную |
| `response_time_promise` | "30 минут" | Обещание клиенту |
| `min_order_amount` | 0 | Мин. сумма заказа |

---

### Лист 4: `analytics` (простая аналитика)

Авто-формулы считают по листу `orders`:

| Метрика | Формула |
|---------|---------|
| Заказов сегодня | COUNTIFS(date, TODAY()) |
| Выручка за месяц | SUMIFS(total_price, month, MONTH(TODAY())) |
| Топ-3 товара | LARGE + INDEX/MATCH |
| % конверсии | orders / bot_users |

## ЗАВИСИМОСТИ И РИСКИ

| Риск | Вероятность | Митигация |
|------|-------------|-----------|
| Владелец не регистрирует бота вовремя | Средняя | Инструкция готова в день 5, напоминание в день 4 |
| Claude API даёт нерелевантные ответы | Средняя | Тщательная проработка system prompt (день 2) |
| Google Sheets медленно читается | Низкая | Кэшировать каталог в памяти бота на 1 час |
| Клиент задаёт вопрос вне каталога | Высокая | Заготовленный ответ + эскалация на владельца |
| Остатки на сайте устарели | Средняя | Предупреждение в ответе: "Уточните наличие у менеджера" |
