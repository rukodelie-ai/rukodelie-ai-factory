## 4. АРХИТЕКТУРА СИСТЕМЫ

### MVP-архитектура (текущий этап)

```
КЛИЕНТ (Telegram)
      │ сообщение / фото / кнопка
      ▼
TELEGRAM BOT (Python + aiogram)
      │ webhook
      ▼
MESSAGE ROUTER
  /start → приветствие
  /catalog → категории
  текст/фото → AI Seller Handler
      │
      ▼
AI SELLER CORE (Claude API)
  model:       claude-sonnet-4-6
  temperature: 0.3
  max_tokens:  1024
  system:      SYSTEM_PROMPT.md (<system_prompt> блок)
               + <customer_context> (если есть история)
               + каталог (только published товары)
  messages:    история диалога (последние 20 сообщений)
      │                    │
      ▼                    ▼
PRODUCT DATABASE      ORDER MANAGEMENT
(Google Sheets)       (Google Sheets)
  лист: catalog         лист: orders
  лист: settings        лист: analytics
```

### Цикл заказа (9 шагов)

```
new → confirmed → paid → assembling → shipped → delivered → closed
```

| Статус | Действие AI |
|--------|------------|
| `new` | "Заказ принят, менеджер свяжется в 30 мин" |
| `confirmed` | "Заказ подтверждён, ожидает оплаты" |
| `paid` | "Оплата получена ✅ Начинаем сборку!" |
| `assembling` | "Собираем ваш заказ 📦" |
| `shipped` | "Заказ отправлен 🚀 Трек: [номер]" |
| `delivered` | "Всё получили? 😊 [персональная рекомендация]" |
| `closed` | Маркетинговый сценарий через 7 дней |

### Память клиента

В system prompt перед каталогом передаётся XML-блок:

```xml
<customer_context>
  <name>Айгерим</name>
  <total_orders>7</total_orders>
  <last_order_date>2026-05-20</last_order_date>
  <favorite_categories>Стразы Swarovski, Бисер Preciosa</favorite_categories>
  <last_products>Стразы ss16 Crystal AB × 200 шт</last_products>
  <preferred_colors>Crystal AB, коралловый</preferred_colors>
  <order_frequency>раз в 2–3 недели</order_frequency>
  <segment>vip</segment>
</customer_context>
```

Сегменты: `new` (0 заказов) / `regular` (1–5) / `loyal` (6–15) / `vip` (16+)

## 6. КЛЮЧЕВЫЕ РЕШЕНИЯ

| # | Решение | Обоснование |
|---|---------|-------------|
| 1 | **AI Seller = персона "Алина"**, консультант 5 лет | Человекоподобная персона даёт конверсию выше безымянного бота |
| 2 | **System prompt в тегах `<system_prompt>`** | Документация команды отделена от промпта API; безопасно обновлять |
| 3 | **claude-sonnet-4-6, temperature=0.3** | Точное следование правилам + естественная речь; 0.0 звучит роботизированно |
| 4 | **Язык определяется автоматически** | Вопрос о языке ломает первый контакт |
| 5 | **Хранилище заказов — Google Sheets (MVP)** | Нет деплоя БД; владелец видит без технических знаний; заменяется в v2 |
| 6 | **Память клиента через XML-блок** | Claude хорошо парсит структурированный XML; опционален — работает без него |
| 7 | **Сегменты: new/regular/loyal/vip** | 4 уровня достаточно для персонализации |
| 8 | **sale_mode = both** для страз и лент | Продаём и поштучно и упаковкой; товар не снимается при stock < pack_qty |
| 9 | **Маркетинговый сценарий: 1 за диалог** | Агрессивный маркетинг разрушает доверие |
| 10 | **Передача менеджеру < 20% обращений** | Менеджер — исключение, не правило |
| 11 | **Только `status = published` видно AI** | Черновики и снятые товары не существуют для клиента |
| 12 | **Каталог передаётся в system prompt** | Не через RAG/embeddings (MVP); достаточно для 50–200 товаров |
| 13 | **RAM dict для истории диалогов** | SQLite добавляет 57% кода при нулевом преимуществе до 100 сессий; при рестарте история сбрасывается — допустимо для MVP |

## 11. ВАЖНЫЕ ПРАВИЛА ПРОЕКТА

### Правила AI Seller (неизменяемые)

1. **Только published товары.** `status ≠ published` → товар не существует для клиента.
2. **Автоопределение языка.** Не спрашивать клиента о языке.
3. **Один вопрос за раз.** Никогда не задавать несколько уточнений подряд.
4. **Всегда называть цену и наличие.** Клиент не должен спрашивать дважды.
5. **Не давать скидок.** Только из прайса. Оптовые условия — менеджер.
6. **Не называть реквизиты.** Оплата только через менеджера.
7. **Не обсуждать конкурентов.** Ни хорошо, ни плохо.
8. **Эскалация < 20% обращений.** Менеджер — исключение, не правило.
9. **Маркетинг: 1 сценарий за диалог.** Не применять при жалобах и оформлении.
10. **Карточка товара — строго по шаблону.** Не длиннее 8 строк.

### Правила работы с каталогом

- `sale_mode = both`: продаём и поштучно, и упаковками. Товар НЕ снимается если stock < pack_qty.
- `stock` хранится в базовых единицах (`base_unit`): стразы — в штуках, ленты — в метрах.
- Расчёт полных упаковок: `FLOOR(stock / pack_qty)`. Остаток = `stock MOD pack_qty`.
- Пряжа: всегда называть состав, метраж, рекомендуемые спицы/крючок.
- Стразы: всегда называть размер (ss), способ крепления, цену за шт. И за упаковку.
- Бегунки/молнии: обязательно уточнять номер (N3/N5/N8) — должны совпадать.

### Правила разработки фабрики

- **Automation First:** любой повторяющийся процесс автоматизируется.
- **Single Source of Truth:** catalog_v1.csv — единственный источник данных о товарах.
- **Data-Driven:** каждое решение опирается на данные (Analytics Agent в Этапе 2).
- **Owner-in-the-Loop:** владелец получает недельные дайджесты, участвует только стратегически.
- **Quality over Speed:** лучше меньше, но правильно.
