## 5. РЕШЕНИЯ, ПРИНЯТЫЕ СЕГОДНЯ

| # | Решение | Обоснование |
|---|---------|-------------|
| 1 | **System prompt разделён тегами `<system_prompt>`** — документация команды отделена от промпта API | Предотвращает случайную передачу мета-информации в API; удобно для обновлений |
| 2 | **Claude model: `claude-sonnet-4-6`, temperature: `0.3`** | temperature=0.3 даёт естественную речь при строгом следовании правилам; 0.0 делает ответы роботизированными |
| 3 | **Язык определяется автоматически — не спрашиваем клиента** | Вопрос о языке ломает первый контакт; автодетекция по кириллице/латинице работает точно |
| 4 | **Английское имя "Alina" — без изменений** | Имя интернациональное, не требует адаптации; единообразие бренда |
| 5 | **Память клиента — XML-блок `<customer_context>` перед каталогом** | Claude хорошо парсит структурированный XML в system prompt; блок необязателен — работает без него |
| 6 | **Сегменты клиентов: new / regular / loyal / vip** | 4 уровня достаточно для персонализации без избыточной сложности |
| 7 | **Полный цикл заказа: 7 статусов** (`new → confirmed → paid → assembling → shipped → delivered → closed`) | Покрывает весь операционный цикл магазина; AI уведомляет на каждом переходе |
| 8 | **Маркетинговые сценарии — не более 1 за диалог** | Агрессивный маркетинг разрушает доверие; 1 предложение увеличивает средний чек без раздражения |
| 9 | **Pinterest-фото → предложить полный набор** | Клиент с Pinterest уже знает что хочет сделать; полный набор = выше средний чек и меньше повторных запросов |
| 10 | **Хранилище заказов — Google Sheets (MVP)** | Нет кода синхронизации, нет деплоя БД; владелец видит заказы без технических знаний; заменяется на нормальную БД в v2 |
