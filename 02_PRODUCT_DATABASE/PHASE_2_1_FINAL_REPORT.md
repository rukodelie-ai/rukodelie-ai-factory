## Цель этапа

Построить первую производственную базу знаний о товарах (AI Product Knowledge) на пилотной
категории «Пряжа», используя исключительно данные, уже существующие в 1С, через уже
опубликованный OData-интерфейс — без создания новой архитектуры, без записи в 1С, без AI-контента
на этом этапе.

## Какие ошибки были обнаружены и как устранены

1. **HTTP 500 при добавлении лишних полей в `$select`** (`Recorder`, `LineNumber`, поля резерва) —
   устранено сужением `$select` до минимального рабочего набора полей.
2. **Ошибка `AUTOORDER`** при фильтрации `AccumulationRegister_СвободныеОстатки_RecordType`,
   `InformationRegister_ЦеныНоменклатуры_RecordType` и `Catalog_iq_ИзображенияХарактеристик` по
   `Номенклатура_Key` / `Характеристика_Key` / `Склад_Key` — **первоначально ошибочно
   диагностирована как серверная проблема публикации OData**. Контрольная проверка тем же
   Python-кодом, что использовался успешно 2026-07-09, показала: запрос отрабатывает нормально.
   **Установленная причина:** `curl --data-urlencode` кодирует пробел как `+`
   (`application/x-www-form-urlencoded`), а `$filter` в OData требует `%20`. 1С не разворачивает `+`
   обратно в пробел внутри `$filter`, из-за чего выражение `Номенклатура_Key eq guid'...'` ломается
   на этапе разбора. Python-код проекта использует `urllib.parse.quote`, кодирующий пробел как `%20`
   — поэтому работал всегда, включая сегодня.
3. **Разные форматы записи кода/названия цвета в 1С** (`"код - название"`, `"код, название"`,
   `"(код)"` без названия) — устранено: парсер распознал 3 формата, итоговое распознавание —
   6171 из 6173 записей (99.97%). 2 записи с составным кодом через `/` (`"197/237"`) не распознаны
   автоматически, сохранены как есть без потери данных.

## Окончательное архитектурное решение

Зафиксировано как официальные решения проекта:

- ✅ **OData признана рабочим источником данных.** Ошибки прошлых сессий были ошибками клиента
  (`curl`), не сервера 1С.
- ✅ **Python `urllib` — утверждённый способ работы с OData** для этого проекта
  (`urllib.parse.urlencode(params, quote_via=urllib.parse.quote)`).
- ✅ **`curl` для работы с OData этого проекта больше не использовать** — установленная причина
  ошибок, зафиксировано как правило.
- ✅ **AI Seller использует общий остаток товара по всем виртуальным складам**, не разбивку по
  складам.
- ✅ **Внутреннее распределение остатков по складам используется только для внутренних процессов
  компании** (ревизия, аналитика, автоматическое резервирование и перемещение) — не для клиента.
- ✅ **Покупателю названия/остатки по конкретным виртуальным складам никогда не показываются.**
- ✅ **Категория «Пряжа» полностью обработана** — 204 товара, все найденные цветовые варианты.
- ✅ **Создана первая производственная база знаний AI Product Knowledge.**
