## 6. Alize Puffy — проверка гипотезы о петлевой пряже (п.5 задания)

Реальные данные подтверждают вашу гипотезу:

- **`Состав` = «100% микрополиэстер»**, **`recommended_needles`** и **`recommended_hook`** — **пустые**. Это не пробел данных, а корректный факт: у Puffy спицы/крючок физически не нужны (петли уже готовы).
- **`Длина` = «9м»** — подтверждён факт малого метража на моток (типично для толстой петлевой пряжи).
- **`characteristics_mode` = «ИндивидуальныеДляНоменклатуры»** — то есть у Puffy механизм характеристик (вариантов) включён, но извлечь конкретные цвета через OData не удалось (см. раздел «Что невозможно получить автоматически»).
- Реальные остатки по линейке: Puffy — 852 (склад: Магазин 42, Интернет-магазин 678, Wolt 132), Puffy Color — 75, Puffy Fine — 327, **Puffy Fine Color — 0 (нет в наличии)**, Puffy More — 2 (почти распродан).
- Реальные цены Puffy: розница 1050₸, Kaspi 1208₸ (+15% — совпадает с уже принятой в проекте бизнес-логикой наценки Kaspi), Halyk 1180₸, интернет-магазин 1050₸.

**Вывод для будущей структуры карточки (Phase 2, не сейчас):** поля `recommended_needles`/`recommended_hook`/`color_base_field` не universal — их пустота у части товаров осмысленна (петлевая пряжа, товар без выраженного «базового» цвета), а не признак плохих данных. Стандарт карточки должен различать «поле неприменимо» и «поле не заполнено» — это два разных состояния качества данных.

## 7. Что невозможно получить автоматически (в этой сессии)

| Что | Почему |
|---|---|
| **Цвета/варианты (характеристики) по каждому товару** | `Catalog_ХарактеристикиНоменклатуры` содержит 146 019 записей на всю базу 1С. Поле `Owner` (ссылка на товар-владельца) — нефильтруемое: `$filter=Owner eq '<guid>'` возвращает пустой результат даже для заведомо существующей записи (проверено на реальном примере — прямой запрос по ключу находит запись, фильтр по тому же значению Owner — нет). Похоже на ограничение конкретной версии OData-интерфейса 1С. Нужна консультация Артёма — либо другой синтаксис фильтра, либо серверная доработка. |
| **Штрихкоды** | Сущность `InformationRegister_ШтрихкодыНоменклатуры` найдена, но не запрашивалась в этой сессии — вне таймбюджета этапа, технически то же самое ограничение по фильтрации может повториться. |
| **Дополнительные реквизиты** | Механизм в 1С есть (`ДополнительныеРеквизиты`), но по ранее проверенной выборке (см. `ODATA_PRODUCT_KNOWLEDGE_MVP_RESEARCH.md`) — пуст. Не перепроверялся отдельно для пряжи в этой сессии. |
| **Бинарное содержимое фото** | Ссылка на файл (`photo_file_guid`) есть у всех 204 товаров, но получение самого изображения через `$value` не проверялось — та же нерешённая техническая задача из предыдущего исследования. |
| **«Текущая» цена/остаток как отдельное поле 1С** | Не существует как факт — вычислено агрегацией 322 849 (цены) и 1 104 522 (остатки) сырых записей движений. Это не поле 1С, а расчёт поверх сырых данных, необходимый только для того, чтобы представить «одна строка — один товар» без потери сути (не аналитика, не нормализация содержания). |
| **Архивная ветка «Старая ПРЯЖА» (4515 товаров)** | Найдена и посчитана, но НЕ выгружена по полям — 99.3% помечены на удаление в 1С, полная выгрузка потребовала бы кратно больше времени ради данных, которые уже не в продаже. GUID-ы папок сохранены, доступны для отдельного запроса, если понадобится. |
